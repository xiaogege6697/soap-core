# SOAP-Core v0.5 Implementation Notes

日期：2026-06-26

## 1. 本轮目标

v0.5 已从 CLI/report 外壳推进到算法落地。当前状态：

1. Simplex 最近邻加权预测已实现（`soap/prediction/simplex.py`）。
2. S-Map 局部线性预测已实现（`soap/prediction/smap.py`，基于 numpy）。
3. Recurrence 诊断已实现（`soap/metrics/recurrence.py`）。
4. 三个模块已接入 CLI 与 report，默认 `--predictor none` / 不启用 `--recurrence` 的旧命令不受影响。

演进说明：v0.5 的初版只固化 CLI/report 外壳与“可选模块发现”约定（见第 3 节），随后算法模块按 CLI 已适配的函数名落地，外壳与实现在 API 层面对齐，**CLI 入口与参数未改动**，保持向后兼容。

## 2. CLI 参数

参数沿用外壳版本，未新增、未改名：

```text
--predictor none|simplex|smap
--recurrence
--recurrence-radius <float>
--recurrence-quantile <float>
--smap-theta <float>
--neighbors <int>
```

当前生效语义：

- `--predictor simplex`：调用 `simplex_predict` + `prediction_skill`。
- `--predictor smap`：调用 `smap_predict` + `smap_skill`。
- `--recurrence`：调用 `recurrence_matrix` + `recurrence_summary`。
- `--neighbors`：Simplex / S-Map 近邻数（默认 10）。
- `--smap-theta`：S-Map 局部线性权重（默认 2.0）。
- `--recurrence-quantile`：自动 radius 的距离分位（默认 0.1）。
- `--recurrence-radius`：显式 radius，缺省时按 quantile 自动估计。

## 3. 模块与函数签名

CLI 用 `importlib` 直接发现子模块，函数签名已固定：

### `soap.prediction.simplex`

- `simplex_predict(embedding, target=None, neighbors=None) -> list[list[float]]`：`neighbors=None` 时默认 `d+1`；对每个状态取近邻，按指数权重加权其下一状态。
- `prediction_skill(predicted, actual) -> {rmse, mae, skill}`：`skill = 1 - mse / mean_baseline_mse`，1.0 为完美，`≤0` 表示不优于均值基线。

### `soap.prediction.smap`

- `smap_predict(embedding, theta=2.0, neighbors=None) -> list[list[float]]`：numpy 实现；加权局部最小二乘，近邻不足或秩亏时退化为加权平均。
- `smap_skill(predicted, actual) -> {mae, rmse, rho}`：`rho` 为展平后的 Pearson 相关。

### `soap.metrics.recurrence`

- `recurrence_matrix(points, radius=None, quantile=0.1) -> list[list[int]]`：`radius` 缺省时取全两两距离的 `quantile` 分位作为阈值。
- `recurrence_summary(matrix) -> {recurrence_rate, determinism_proxy, average_diagonal_length_proxy}`：基于对角线段长度估计确定性。

容错保留：若上述模块缺失或调用抛异常，CLI 将异常写为 `status: unavailable` 并附原因，不影响默认分析链路。

包级公开导出（本轮补齐）：

- `soap.prediction`：`simplex_predict`, `prediction_skill`, `smap_predict`, `smap_skill`
- `soap.metrics`：`estimate_delay_ami`, `estimate_embedding_dimension_fnn`, `recurrence_matrix`, `recurrence_summary`

## 4. 报告输出

`report.md` 固定包含两个区块：

```text
## Prediction Diagnostics
## Recurrence Diagnostics
```

状态语义：

- `--predictor none`：`status: disabled`, `predictor: none`
- 未传 `--recurrence`：`status: disabled`, `enabled: False`
- 跑通时 `status: ok`，并附加各诊断字段：
  - simplex：`rmse`, `mae`, `skill`
  - smap：`mae`, `rmse`, `rho`
  - recurrence：`recurrence_rate`, `determinism_proxy`, `average_diagonal_length_proxy`, `matrix_size`, `radius`, `quantile`

## 5. 验收结果

Simplex + recurrence（`synthetic_cyclic`，window embedding，pca）：

```bash
PYTHONDONTWRITEBYTECODE=1 .venv/bin/python -m soap.cli analyze examples/synthetic_cyclic.csv --embedding window --method pca --predictor simplex --recurrence --neighbors 12 --recurrence-quantile 0.1 --clusters 3 --output-dir outputs_claude_v05_simplex --report-dir reports_claude_v05_simplex
```

终端输出：


解读：`synthetic_cyclic` 是确定性周期序列，PCA 识别 `d*=2`（周期相位环）；simplex `skill≈1.0`、smap `rho=1.0` 说明一步预测近乎完美；recurrence `determinism_proxy≈0.998` 印证确定性结构，`recurrence_rate≈0.103` 贴近 `quantile=0.1` 的自动阈值设定。

## 6. 边界

1. simplex 为纯 Python 实现，smap 用 numpy；二者均为全库近邻搜索，未做空间索引加速。
2. `recurrence_matrix` 是 `O(n²)` 全两两距离，300 点秒级可完成，大规模序列需优化。
3. `smap_skill` 的 `rho` 在预测方差为 0 时置 0（见实现），但确定性近完美数据上仍可读到 `rho=1.0`。
4. 本轮不改 `soap/prediction/transition.py`，不重构维度/嵌入模块，CLI 入口与参数保持兼容。
5. 报告未生成 recurrence plot 图像，仅输出文本 RQA 代理量；可视化留待后续。
6. 仍未输出 AMI curve / FNN curve / prediction skill curve 的完整曲线，只输出最终诊断标量。
