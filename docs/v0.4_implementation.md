# SOAP-Core v0.4 Implementation Notes

日期：2026-06-26

## 1. 本轮目标

v0.4 的目标是把 SOAP-Core 从“直接使用观测窗口”推进到“相空间重构”：

1. Takens delay embedding。
2. AMI 自动估计 delay。
3. FNN 自动估计 embedding dimension。
4. CLI 支持 `window` 与 `takens` 两条 embedding 路径。

## 2. 管家并行执行

本轮按管家工作流执行：

- 临时工 A：实现 `soap/embedding/takens.py`。
- 临时工 B：实现 `soap/metrics/ami.py`。
- 临时工 C：实现 `soap/metrics/fnn.py`。
- 临时工 D：只读审查 v0.4 CLI / report / smoke test 设计。
- 主线程：接口集成、报告扩展、验收、文档记录。

这个分工有效，因为三个算法模块可以按文件独立开发，CLI/report 属于系统接口层，必须由主线程统一把关。

## 3. CLI 设计

新增参数：

```text
--embedding window|takens
--delay auto|<int>
--embedding-dim auto|<int>
--max-delay <int>
--max-embedding-dim <int>
```

语义边界：

- `--window-size`：旧路径的连续窗口拼接。
- `--delay`：Takens delay `tau`。
- `--embedding-dim`：Takens embedding dimension `m`。
- `--max-dim`：下游 effective dimension / PCA scoring 的搜索上限。

关键决策：不复用 `--max-dim` 作为 FNN 的 `max_dim`，避免理论语义混乱。

## 4. 验收结果

兼容路径：

```bash
PYTHONDONTWRITEBYTECODE=1 .venv/bin/python -m soap.cli analyze examples/synthetic_cyclic.csv --embedding window --window-size 1 --method variance --max-dim 8 --clusters 3 --output-dir outputs_v04_compat --report-dir reports_v04_compat
```

结果：

```text
Optimal effective dimension: 5
Dimension method: variance
Embedding method: window
```

Takens 自动路径：

```bash
PYTHONDONTWRITEBYTECODE=1 .venv/bin/python -m soap.cli analyze examples/synthetic_lorenz.csv --embedding takens --delay auto --embedding-dim auto --max-delay 50 --max-embedding-dim 8 --method pca --clusters 3 --output-dir outputs_v04_smoke --report-dir reports_v04_smoke
```

结果：

```text
Optimal effective dimension: 3
Dimension method: pca
Embedding method: takens
Takens delay: 15
Takens embedding dimension: 1
```

解释：Lorenz 示例本身是三变量观测，FNN 给出 `m=1` 表示当前观测空间已经足够表达主要状态，不需要额外 delay block；PCA 后仍识别 `d*=3`。

## 5. 已知边界

1. AMI 对多变量数据当前默认使用第一观测列估计 delay。
2. FNN 当前返回 embedding dimension，但未向报告输出完整 ratio curve。
3. Takens 自动路径已经可运行，但还没有和 Simplex / S-Map 预测头结合。
4. `.venv` 当前未安装 `pytest`；`pyproject.toml` 已声明 `dev = ["pytest"]`，后续可安装 `.[dev]` 后跑完整测试。
