# SOAP-Core v0.7.2 Representation Metrics Input Schema
日期：2026-06-27
定位：定义 representation metrics 的真实/半真实数据输入 schema，为 v0.7.3 多层 benchmark 与真实模型接入铺路。toy benchmark 阈值不固化进 CLI。

## 1. 为什么需要 schema
v0.7/v0.7.1 的 representation 指标（effective_rank/collapse_score）只在 torch toy benchmark（单层最后隐藏层）验证。真实模型（HuggingFace transformers 等）的 representation 是多层、高维、按任务变化的。需要一个明确 schema 让 representation 数据有统一输入格式，便于 v0.7.3 多层 benchmark 和真实模型接入，同时避免把 toy 阈值过早固化。

## 2. 字段定义
| 字段 | 必需 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| timestamp | 必需 | str(step_i)/int | 训练步或时间戳，单调递增，作排序基准 |
| effective_rank | 必需 | float(≥1) | 隐藏层激活协方差的 entropy effective rank |
| representation_variance | 必需 | float(≥0) | 激活总方差 |
| collapse_score | 必需 | float(≥0) | 1/effective_rank proxy |
| layer_name | 可选 | str | 层标识（如 encoder.layer.11），多层时必需 |
| activation_mean | 可选 | float | 激活均值，辅助诊断 |
| activation_std | 可选 | float(≥0) | 激活标准差 |
| sample_count | 可选 | int(>0) | 计算表示的样本数，可复现性 |

## 3. 必需 vs 可选原则
- 必需字段是 direct detection（detect_representation_collapse）依赖的 effective_rank / collapse_score，加上排序用的 timestamp；representation_variance 保留必需（v0.7 已记录，辅助区分）。
- 可选字段：layer_name（单层 toy 可省，多层必需）、activation_mean/std（辅助诊断）、sample_count（可复现性）。

## 4. 多层 representation 原则
- 真实模型每层 representation 几何不同（浅层 vs 深层 collapse 形态不同）。
- 多层时应分别检测每层 collapse，再聚合（如任一层 collapse → 风险，或按层加权聚合）。
- layer_name 字段区分层。v0.7.3 多层 benchmark 将验证此原则。

## 5. toy threshold vs 真实校准
- v0.7.1 默认阈值（collapse_score>0.9 且 effective_rank<1.5）只适用于 torch toy benchmark（CPU 小 MLP）。
- 真实模型需按层、按任务重新校准：不同模型/任务的 effective_rank 量级、collapse 形态不同，不能直接套用 toy 阈值。
- adapter 的 thresholds 参数支持覆盖，真实接入时应基于校准数据设定，不把 toy 阈值写死进 CLI。

## 6. 真实数据来源（预期）
- HuggingFace transformers：用 forward hook 截取隐藏层激活，计算 effective_rank/collapse_score。
- 自定义训练：在训练循环中对验证集前向，提取各层激活。
- 输出符合本 schema 的 CSV，供 representation_adapter 直接检测。

## 7. 边界
- schema 是输入格式约定，不绑定具体模型框架。
- collapse_score=1/effective_rank 是 proxy，真实 collapse 度量可能需更严格指标（如 NC1、CCA）。
- 本版只定义 schema，不改 CLI/核心 SOAP，不固化阈值；下一版 v0.7.3 做多层 benchmark，v0.8 才统一 CLI 封装。
