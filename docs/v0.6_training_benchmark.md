## 4. v0.6.4 修复与指纹演变（关键）
- v0.6.3 mode_collapse 的 grad_norm 在坍缩相遗漏 abs()，末步出现负值（-0.026）。v0.6.4 修复为 abs(...)，grad 全部 >=0（min=0.000109）。
- abs() 不消费随机数，故 mode_collapse 的 loss/val 列数值与 normal/diverging/overfit 全部不变（normal 末行 loss 仍=0.12814119）。
- 但 grad 列从"正负振荡"变成"单调衰减到 0"，改变了状态空间结构：mode_collapse 的 d* 从 v0.6.3 的 3 降到 v0.6.4 的 1。
- 结论修正：v0.6.3 因 grad bug 误判 mode_collapse 为"第三类（d*=3 混合指纹）"；修复后真实指纹是 d*=1（坍缩型）。数据语义 bug 扭曲了 SOAP 指纹，证明数据正确性对结论关键。

## 5. 修正后的 Taxonomy（三类 + 二级分类）
- **normal（基线）**：d*=2，det≈0.52，skill≈0.56，next 分散——健康收敛。
- **坍缩型失稳（d*↓ + det↑ + skill↑）**：状态空间坍缩到低维 + 过确定性。
- **扩张型失稳（d*↑ + det 正常）**：状态空间扩张。

四类落位：
- normal → 基线
- diverging → 坍缩型（d*=1，det=0.96，grad 爆炸，loss 爆炸）
- mode_collapse → 坍缩型（d*=1，det=0.86，grad 衰减，loss 平台）
- overfit → 扩张型（d*=3，det=0.59，train 降 val 升）

## 6. 二级分类：坍缩型内部再分（v0.6.3 预判在 v0.6.4 印证）
坍缩型失稳（d*↓）含两种，由 grad_norm 与 loss 方向再区分：
- **divergence 型**：grad_norm 爆炸 + loss 爆炸（动力学坍缩到异常确定性通道）
- **collapse 型**：grad_norm 衰减到 0 + loss 平台化（低动态多样性/平台化坍缩）

含义：SOAP 能在失稳内部做二级分类。这正是 v0.6.3 立项时的预判，v0.6.3 因 grad bug 未能呈现，v0.6.4 修复后成立。

## 7. 哪些指标有区分度（修正）
- **d*（2/1/3/1）**：区分扩张（overfit=3）与坍缩（diverging/mode_collapse=1）与基线（normal=2）。
- **determinism/skill**：挑出过确定性（diverging 最高，mode_collapse 次高）。
- **grad_norm + loss 行为**：坍缩型内部区分 divergence vs collapse 的关键（SOAP 内部指标 d* 与 det 在两者上方向一致，需结合原始 grad/loss 绝对行为）。
- **组合（d* 方向 + det 是否飙升 + grad/loss 行为）**：完整分类依据。recurrence_rate 无区分。

## 8. 边界
- 基于 4 种合成数据，真实 training run 验证前不过度外推。
- mode_collapse 用通用 loss/grad/val 字段近似，真实 representation collapse 的 effective rank/embedding similarity 字段未覆盖。
- recurrence_rate 无区分（quantile 阈值）。
- 教训：数据语义正确性（如 grad_norm 非负）会实质影响 SOAP 指纹（d* 3→1），合成与真实数据都须先校验字段语义。

## 9. 结论与下一步
- **结论（基于合成数据）**：SOAP 能提取训练过程结构指纹，失稳可分为坍缩型（d*↓+过确定性，内含 divergence 与 collapse 二级）与扩张型（d*↑，overfit）。修复 grad 语义后 taxonomy 更清晰且自洽。
