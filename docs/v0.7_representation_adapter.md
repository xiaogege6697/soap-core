# SOAP-Core v0.7.1 Representation Geometry Adapter

日期：2026-06-27

定位：把 v0.7 的 representation 几何指标正式化为 direct detection adapter，不依赖 SOAP analyze pipeline。

## 1. 为什么 direct detection

v0.7 发现 effective_rank / collapse_score 能明显拉开 mode_collapse，但 SOAP 动力学指纹（skill / det / next）对 mc / normal / overfit 仍不分。结论：representation 失稳（mode_collapse）的区分信号在几何指标本身，不在曲线动力学。因此 representation collapse 走 direct detection（直接从指标判定），而非强行依赖 SOAP pipeline。项目由此确立两层诊断架构：动力学失稳（divergence）用 SOAP 状态空间，representation 失稳（mode_collapse）用几何指标直判。

## 2. 输入字段

repr-enhanced CSV（v0.7 `torch_repr_runs.py` 产出）。adapter 只读取三列：`effective_rank`、`representation_variance`、`collapse_score`（标准 `loss` / `val_loss` / `grad_norm` / `learning_rate` / `train_val_gap` / `output_variance` 也存在但 adapter 不使用）。

## 3. summary 输出

`summarize_representation_metrics(csv_path)` 对三个指标列各返回：`start` / `end` / `mean` / `min` / `max` / `drop_ratio`。`drop_ratio = (start - end) / start`，度量训练中下降比例（见第 4 节：`drop_ratio` 不参与判定）。

## 4. detection 规则

`detect_representation_collapse(summary, thresholds=None)` 的判定逻辑：

- **规则 A（主，生效）**：`collapse_score_final > 0.9` 且 `effective_rank_final < 1.5` → collapse。collapse 以 final 值（eff_rank 接近 1、collapse_score 高）为准。
- **规则 B（已移除）**：曾用 `effective_rank drop_ratio > 0.7` 作辅助规则，实测 normal（drop_ratio 0.79）/ overfit（0.76）训练中 eff_rank 自然聚焦下降也触发，误判严重（四类全判 collapse），故移除。教训：eff_rank 在所有正常训练中都会下降，drop_ratio 不是 collapse 特征。
- 默认阈值 `collapse_score_min=0.9`、`effective_rank_max=1.5` 是 toy benchmark proxy，非通用标准；`thresholds` 参数可覆盖。

## 5. toy benchmark 结果（四类 detect）

| 类型 | status | collapse_score_final | effective_rank_final | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| normal | ok | 0.6535 | 1.5303 | 不触发（cs<0.9） |
| divergence | collapse | 10000（cap） | 0.0000 | 爆死后表示消失，representation 层判 collapse |
| overfit | ok | 0.3039 | 3.2907 | 不触发（cs<0.9） |
| mode_collapse | collapse | 0.9983 | 1.0017 | 触发（cs>0.9 且 er<1.5） |

**两层互补**：divergence 在 SOAP 动力学层（skill 崩）和 representation 层（表示消失）都触发；mode_collapse 只在 representation 层触发（SOAP 动力学正常）；normal / overfit 两层都不触发。结合两层可区分 divergence vs mode_collapse。

## 6. 边界

- **toy proxy 阈值**：0.9 / 1.5 基于 CPU 小 MLP toy benchmark，真实大模型 embedding 需重新校准。
- `collapse_score = 1/effective_rank` 是 proxy，非严格 collapse 度量。
- divergence 爆死后 collapse_score 被 `_safe_float` cap 到 1e4，属极端，真实训练应配合动力学层判定。
- adapter 不修改 CLI / 核心 SOAP，是独立 direct detection 模块。

## 7. 测试

`tests/test_representation_adapter.py` 共 5 用例：normal 不判 collapse / mode_collapse 判 collapse / overfit 不误判（验证规则 B 移除）/ 缺字段报 `ValueError` / summary 字段完整。pytest 全过（5 passed）。
