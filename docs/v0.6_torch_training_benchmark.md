# SOAP-Core v0.6.8 Torch Training Benchmark
日期：2026-06-27
一句话定位：用真实 PyTorch 训练过程生成四类 training logs，验证 SOAP 失稳 taxonomy 能否从 synthetic / realish 数据迁移到真实训练动力学。

## 1. 为什么这一步不同于 synthetic / realish
- synthetic（v0.6.1~0.6.4）：手写数学曲线（exp 衰减、递归放大），loss/grad 是预设函数，刻意为每类构造特定 d* 方向。
- realish（v0.6.6）：合成 normal 数据套 W&B 字段名壳，本质仍是 synthetic，曲线形态不变。
- torch 真实训练（v0.6.8）：真实 optimizer.forward/backward/grad_norm/loss 动力学，曲线形态由真实优化轨迹决定，不再手写。这是 synthetic → realish → real training process 的正确过渡。

## 2. 训练任务设置
- 自生成非线性回归数据集：y = 2*sin(x1) + 0.5*sum(x^2) + noise，X 为 6 维，无外部数据集下载。
- 共享 MLP（Linear+ReLU），CPU 轻量训练，每个 run 200~400 step，单 run 几十秒。
- 每 step 记录 loss/val_loss/grad_norm/learning_rate，CSV 首列 timestamp 为 step_{i} 字符串（防 SOAP load_csv 把数值 step 当维度）。
- mode_collapse 额外记录 output_variance 到单独 CSV（不进 SOAP 标准分析）。
- 脚本：soap/apps/training/torch_runs.py。

## 3. 四类 run 生成机制
- normal：MLP[32,16]，Adam lr=1e-3，weight_decay=1e-4。合理训练，loss/val 平滑下降。
- divergence：MLP[32,16]，mini-batch(batch=16) + SGD lr=0.5。关键点——full-batch SGD 在准凸 MSE 上只有“收敛/爆死”两态，无法持续震荡；引入 mini-batch 梯度噪声 + 大 lr 才能产生真实的持续震荡爆炸（loss 爆到 1756，grad 爆到 4736）。这是真实训练 divergence 的经典成因。
- overfit：30 个训练样本 + 宽 MLP[128,64] + Adam lr=1e-3 + weight_decay=0，400 step。train loss 拟合到 1e-5，val loss 停在 0.75，generalization gap 巨大。
- mode_collapse：MLP[64,32] + Adam lr=1e-3 + 极大 weight_decay=5.0。强 L2 把权重拉向 0，loss 停在非零平台（6.x 不降），output_variance 被压到 0.06（远小于 normal）。这是真实训练动力学中的低动态多样性 proxy（非 GAN mode collapse，但符合 v0.6.8 要求）。
- 调参教训：divergence 经历 full-batch SGD 0.5（爆51后回落）、Adam 0.1（反而纯收敛）、SGD 1.0（第一步爆 inf 被 cap 成 1e4 死常数）多次失败，最终 mini-batch 方案才产出有信息量的真实震荡。

## 6. 迁移结论（哪些成功、哪些失败）
1.  **动力学失稳检测：迁移成功**。divergence 是四类中唯一被 SOAP 清晰识别的：skill 崩溃（-0.004）、det 骤降（0.216）、rmse 爆炸（1.002）、avg_diag 骤降（2.11）、吸引子改变（A1 主导）。SOAP 的状态空间方法对“曲线动力学层面的失稳（震荡/爆炸/不可预测）”敏感且有效。
2.  **语义性失稳分类：迁移失败**。normal / overfit / mode_collapse 三者的 SOAP 指纹几乎相同（skill≈1，det≈1，next A2=1.0），无法区分。根因：三者的训练曲线都“平滑、单调、可预测”，它们的失稳是业务语义层面（overfit 的 train-val 泛化 gap、mode_collapse 的输出分布坍缩），而非曲线几何动力学层面。SOAP 标准 4 字段（loss/val_loss/grad_norm/lr）pipeline 不捕捉这些语义信号。
3.  **d* 方向不迁移**。synthetic 手写曲线刻意构造的 d* 方向（divergence/mode_collapse 坍缩→d*↓，overfit 扩张→d*↑）在真实数据上反向或错位。真实曲线的 d* 反映的是曲线的“有效维度”，不是 taxonomy 的“坍缩/扩张”语义。
4.  **v0.6.6 realish 的“完美迁移”是假象**。realish 本质是 synthetic normal 数据套 W&B 字段名壳，曲线形态与 synthetic 完全相同，故 SOAP 指纹一致；这并不构成“真实迁移”的证据。只有真实训练动力学数据（v0.6.8）才能检验迁移性，而检验结果是：仅动力学失稳层面迁移成立。

## 7. 边界说明（重要）
- 这是“真实训练过程日志”，但仍是受控 toy training（CPU 小模型、自生成回归数据），不等于真实大模型训练。
- 若 taxonomy（动力学失稳部分）成立，只能说明 SOAP 从手写曲线迁移到真实 optimizer 动力学；不能外推到所有真实训练场景。
- overfit/mode_collapse 不可区分，可能部分受 toy 设置影响（如 overfit 的 val 未单调上升、mode_collapse 的 grad 未衰减），但核心结论——平滑可预测曲线的语义失稳难被几何方法区分——预计在真实数据上同样成立。
- 下一步仍需真实项目日志（真实 W&B / TensorBoard run）验证。

## 8. 对下一步的启发
- overfit 检测需要 train-val gap 信号（非曲线几何），可考虑把 train_loss 与 val_loss 的差值/比值作为显式特征。
- mode_collapse 检测需要 output/representation variance 信号（v0.6.8 已记录 output_variance 但未纳入 SOAP 分析），可考虑把 output_variance 作为显式维度接入。
- SOAP 标准 pipeline 适合“动力学失稳（震荡/爆炸）”检测；语义性/结构性失稳需扩展信号源。
- 真实 W&B / TensorBoard run 验证是下一阶段，重点验证 divergence 类（动力学失稳）在真实大模型日志中是否同样被 SOAP 识别。
