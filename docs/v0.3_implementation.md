# SOAP-Core v0.3 Implementation Notes

日期：2026-06-26

## 1. 本轮目标

v0.3 的目标不是增加应用层功能，而是把“有效维度搜索”从 baseline 方差列选择升级为可验证的科学计算路径。

核心新增：

1. PCA 状态空间投影。
2. 时间顺序 train/test 切分。
3. one-step predictive scoring。
4. CLI 可选方法 `--method pca`。

## 2. 设计决策

保留旧方法 `variance` 作为默认值，避免历史命令和报告结果突然变化。

新增 `pca` 方法作为科学计算路径：

- reconstruction error：PCA 低维投影后在 test set 上的归一化重构误差。
- prediction error：用 train set 的低维状态学习一步转移，在 test set 上评估。
- complexity penalty：继续惩罚更高维度，避免无限堆维。

时间序列不能随机切分，因此采用前段 train、后段 test。

## 3. 验收结果

baseline 兼容性：

```bash
PYTHONDONTWRITEBYTECODE=1 .venv/bin/python -m soap.cli analyze examples/synthetic_cyclic.csv --max-dim 8 --clusters 3 --method variance --output-dir outputs_v03_variance --report-dir reports_v03_variance
```

结果：

```text
Optimal effective dimension: 5
Dimension method: variance
```

Lorenz PCA：

```bash
PYTHONDONTWRITEBYTECODE=1 .venv/bin/python -m soap.cli analyze examples/synthetic_lorenz.csv --max-dim 8 --clusters 3 --method pca --output-dir outputs_v03_lorenz_pca --report-dir reports_v03_lorenz_pca
```

结果：

```text
Optimal effective dimension: 3
Dimension method: pca
```

cyclic PCA：

```bash
PYTHONDONTWRITEBYTECODE=1 .venv/bin/python -m soap.cli analyze examples/synthetic_cyclic.csv --max-dim 8 --clusters 3 --method pca --output-dir outputs_v03_cyclic_pca --report-dir reports_v03_cyclic_pca
```

结果：

```text
Optimal effective dimension: 2
Dimension method: pca
```

解释：五个相移正弦观测变量来自同一周期相位环，PCA 识别的是低秩几何结构，因此 `d*=2` 比 `d*=5` 更符合“状态空间最小有效表达”。

## 4. 已知边界

1. 当前 PCA 方法适合连续数值时间序列，不适合离散符号系统。
2. one-step predictive scoring 仍是线性预测头，后续需要加入 Simplex / S-Map / ESN。
3. 当前还没有 Takens delay embedding，`window_size` 只是简单拼接窗口。
4. `pytest` 未安装在 `.venv`，本轮使用 compileall、函数脚本和 CLI smoke test 验收。
