# SOAP-Core v0.7.5 phase-aware baseline deviation detection benchmark 科学文档

## 摘要
本文档记录了 SOAP-Core 项目 v0.7.5 版本引入 phase-aware baseline deviation detection 方法后的对比实验结果。实验在 v0.7.4 controlled rank-1 benchmark 基础上，通过增加相位感知的基线（phase-knots）与持久性（persistent）条件，进一步抑制了在预训练 DistilBERT 微调过程中对深层（deep layers）的误报（false alarm, FA）。实验对比了三种方法：A（v0.7.4 静态阈值）、B（phase-knots 基线）及 C（B + persistent），在受控的 induced collapse 和正常（normal）条件下验证了方法的有效性。

## 证据边界
- **Induced collapse 的性质**：本 benchmark 中的 induced collapse 是人工施加的 controlled rank-1 intervention，旨在验证 baseline deviation detection 与传播分析的有效性，**不代表**自然（natural）mode collapse 已被验证。
- **实验对象范围**：实验对象为预训练 DistilBERT 在合成（synthetic）5 类分类任务上的微调（fine-tuning）运行。结论严格限定于该模型、任务及数据规模，**不可外推**到其他真实模型或自然训练动力学。
- **样本量与推断范围**：误报/检出基于小样本（held-out normal 20 runs / held-out induced 3 runs）。Wilson 95% 置信区间（CI）已提供，但**不可外推**为总体百分比。
- **FPR 比较限制**：三种方法的误报率（FPR）定义不同（per-layer / run-level any_layer / persistent），因此**不可混用**。
- **术语边界**：本方法仅可称为 baseline deviation detection，**不宣称** phase-aware effective_rank 能够区分良性 Neural Collapse 与病态 collapse。

## 实验设置
- **模型**：`distilbert-base-uncased`（冻结 embedding + 前 3 个 block，可训练 block 3-5 + 分类头），模型 revision 为 `12040accade4e8a0f71eabdb258fecc2e7e948be`。
- **软件环境**：`transformers 5.12.1`，`torch 2.12.1`，计算设备为 Apple MPS（Metal Performance Shaders）。
- **模型加载**：使用 `from_pretrained` 时设置 `local_files_only=True`。首次运行前需预下载 `distilbert-base-uncased` 到 HuggingFace 缓存。此为可复现性约束（避免联网检查 revision 的网络依赖或超时），**不是**通用安装体验要求。
- **数据**：5 类结构化英文模板（训练集 1600 样本 / 验证集 400 样本 / 探测集 128 样本），`data_seed=0` 固定。
- **种子与条件**：正常运行（normal）使用种子 42-73，诱导崩溃（induced）运行使用种子 42-47。Alpha 调度与 v0.7.4 一致。
- **数据划分**：开发集（dev）为正常种子 42-53 及诱导种子 42-44；保持集（held-out）为正常种子 54-73 及诱导种子 45-47。

## 三方法（A/B/C）
- **方法 A（v0.7.4 静态阈值）**：采用静态逐层（static per-layer）`collapse_score` 的 99 百分位阈值，配合 `any_layer` 逻辑。此为 v0.7.4 基准方法。
- **方法 B（phase-knots 基线）**：使用 8 个相位节点（phase-knots）线性插值计算 `effective_rank` 的中位数（median）与 MAD（Median Absolute Deviation）尺度参数（`MAD_scaled = 1.4826 * MAD`）。引入逐层 MAD 下限（layer floor）与运行级别最大异常值（run-level max anomaly），在开发集正常运行（dev normal 42-53）上通过 leave-one-seed-out 校准得到阈值 `threshold_B = 10.79`（`run_q=0.95`）。
- **方法 C（B + persistent）**：在方法 B 的基础上，增加持久性条件：要求异常在同一层连续出现 ≥3 个 checkpoint 才判定为异常。
- **注**：`collapse_score = 1/eff_rank` 与 `effective_rank` 提供重复信息，因此方法 B/C 仅使用 `effective_rank`。

## 结果

### 误报（FA）与检出（Det）对比

| 方法 | 误报 (FA) - held-out normal (20 runs) | Wilson 95% CI (FA) | 检出 (Det) - held-out induced (3 runs) | Wilson 95% CI (Det) |
| :--- | :--- | :--- | :--- | :--- |
| **A** (static) | 20 / 20 | (0.8389, 1.0000) | 3 / 3 | (0.3085, 1.0000)¹ |
| **B** (phase-knots) | 0 / 20 | (0.0000, 0.1611) | 3 / 3 | (0.3085, 1.0000)¹ |
| **C** (persistent) | 0 / 20 | (0.0000, 0.1611) | 3 / 3 | (0.3085, 1.0000)¹ |

¹ 注：当样本量 n=3 时，Wilson 95% CI 范围宽，强调结果基于极小样本，**不具外推性**。

### Exact McNemar 配对检验（在 held-out normal 20 runs 上）
基于检出 (1) 与未检出 (0) 的配对比较：

| 比较 | n01 (A未检出 B检出) | n10 (A检出 B未检出) | n_discordant | p_value |
| :--- | :--- | :--- | :--- | :--- |
| **A vs B** | 0 | 20 | 20 | 0.000002 |
| **A vs C** | 0 | 20 | 20 | 0.000002 |
| **B vs C** | 0 | 0 | 0 | 1.000000 |

### 阈值 q 敏感性分析
在 `run_q=0.90`（sensitivity q）下，FA 与 Det 结果与主结果（`run_q=0.95`）完全一致：
- 方法 A: FA=20/20, Det=3/3
- 方法 B: FA=0/20, Det=3/3
- 方法 C: FA=0/20, Det=3/3

### 相位基线诊断（dev normal 42-53）
- **冻结层（layer_0-3）**：MAD=0。`effective_rank` 在整个训练过程中保持不变（自洽基线）。
- **可训练层（layer_4-6）**：MAD>0，layer floor 分别约为 0.036、0.078、0.074。`log(effective_rank)` 中位数随 normalized phase 从约 2.25、1.76、1.63 区间下降。

## 解读
- **方法 A 的 20/20 误报**：在当前 DistilBERT synthetic-task benchmark 条件下，静态逐层阈值方法在真实深层 normal 训练的后期完全失效。
- **方法 B/C 的 0/20 观察**：在当前 held-out 20 runs 中观察到 0/20 的误报。
- **可训练层 `log(effective_rank)` 随相位下降**：此现象与训练后期自然表示聚焦（representation focusing）相容，**但不足以证明**是 Neural Collapse。
- **风险改善**：从方法 A 到方法 B，观察到的误报（FA）从 20/20 降至 0/20。
- **分离来源注意**：完美分离（0/20 误报 vs 3/3 检出）部分源于 induced collapse 为 controlled rank-1 intervention，其在数学空间上与 effective-rank 指标对齐。这有利于验证检测器，但也意味着该分离质量**不能直接归因于**检测器对自然 collapse 的判别力。

## 尚未验证
- **泛化性未验证**：跨任务、跨模型、自然 mode collapse 条件下的有效性均未验证。
- **校准依赖性**：phase baseline 阈值（如 `threshold_B=10.79`）必须针对特定模型、层、任务重新校准；当前阈值**不可直接迁移**到其他设置。

## 产品结论
- **推荐默认方法**：推荐 phase-aware 方法 B 作为新的默认方法，替代静态方法 A。
- **持久性条件暂不采用**：方法 C（persistent）在本 benchmark 中未提供额外增益（B vs C p=1.0），为避免增加产品复杂度，**暂不进入**默认路径。
