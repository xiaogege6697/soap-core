# SOAP-Core v0.7.3 Multi-layer Representation Benchmark
日期：2026-06-27
定位：验证 v0.7.2 schema 的多层 representation，检验 mode_collapse 是否首现特定层、三种聚合规则是否误判 normal/overfit。逐层诊断，不输出单一全局结论。

## 1. 为什么多层
v0.7/v0.7.1 只看最后隐藏层（单层）。v0.7.2 schema 定义 layer_name 支持多层。v0.7.3 验证：真实模型 collapse 可能是某层局部现象，单层看不全，需逐层检测 + 聚合规则。

## 2. 多层设置
- 脚本 soap/apps/training/torch_multilayer_repr_runs.py，复用 v0.6.8 训练设置（_make_mlp 两隐藏层），multilayer_activations 提取每个 ReLU 后激活（hidden_0 / hidden_1）。
- 每层每步算 effective_rank / representation_variance / collapse_score。
- CSV 9 字段（timestamp, layer_name, loss, val_loss, grad_norm, learning_rate, effective_rank, representation_variance, collapse_score），每步每层一行。
- multilayer adapter（soap/apps/training/representation_multilayer_adapter.py）复用单层 detect_representation_collapse，按层分组 + 三种聚合。

## 3. 逐层诊断（关键）
| 类型 | hidden_0 (eff_rank / collapse) | hidden_1 (eff_rank / collapse) | 观察 |
| :--- | :--- | :--- | :--- |
| normal | ok (9.73 / 0.10) | ok (1.53 / 0.65) | 两层正常 |
| divergence | ok (2.48 / 0.40) | collapse (0 / 1e4) | 深层爆死后消失 |
| overfit | ok (12.81 / 0.078) | ok (3.29 / 0.30) | 两层正常（表示丰富） |
| mode_collapse | ok (10.06 / 0.099) | collapse (1.00 / 0.998) | 仅深层 hidden_1 坍缩 |

核心发现：mode_collapse 首现特定层（深层 hidden_1），非全层同步。hidden_0（浅层/宽层）eff_rank 10.06 正常，hidden_1（深层/窄层）eff_rank 1.00 完全坍缩。验证了 collapse 是深层局部现象的假设。divergence 同样只在 hidden_1（爆死后深层先消失）。

## 4. 三种聚合对比
| 类型 | any_layer | consecutive (≥2 层) | severity_weighted (均值>0.9) | severity_mean |
| :--- | :--- | :--- | :--- | :--- |
| normal | False | False | False | 0.378 |
| divergence | True | False | True | 5000.2 |
| overfit | False | False | False | 0.191 |
| mode_collapse | True | False | False | 0.549 |

聚合规则：any_layer = 任一层 is_collapse；consecutive = 按层顺序连续 ≥2 层 collapse（本 toy 仅 2 层，即两层都 collapse）；severity_weighted = 各层 collapse_score_final 均值 > 0.9。

## 5. 误判矩阵（聚合质量）
| 类型 | 真实 | any_layer | consecutive | severity_weighted |
| :--- | :--- | :--- | :--- | :--- |
| normal | ok | ✅ ok | ✅ ok | ✅ ok |
| overfit | ok | ✅ ok | ✅ ok | ✅ ok |
| divergence | collapse | ✅ collapse | ❌ 漏判 | ✅ collapse |
| mode_collapse | collapse | ✅ collapse | ❌ 漏判 | ❌ 漏判 |

结论：any_layer 聚合最优，4/4 正确（normal/overfit 不误判，divergence/mc 都检出）。consecutive (≥2 层) 太严：toy 仅 2 层，mc/div 只 1 层 collapse，漏判；在更多层的真实模型可能适用。severity_weighted (均值>0.9) 漏判 mc：hidden_0 正常 (cs 0.099) 拉低均值 (0.549<0.9)，正是 collapse 局部性的反证——全层均值掩盖局部 collapse。

## 6. 单层 adapter 复用
multilayer adapter 直接复用单层 detect_representation_collapse（每层调一次），不另建算法。新增的只是按层分组（summarize_multilayer）+ 聚合逻辑。符合"单层 adapter 可直接复用"要求。

## 7. 负面结果（保留）
- consecutive 聚合在 toy（2 层）完全失效（所有类都 False），不能用于 toy，需 ≥3 层模型才有意义。
- severity_weighted 漏判 mode_collapse（深层局部 collapse 被浅层正常拉低均值）。
- 任何"全层聚合"（均值 / 连续）都会掩盖局部 collapse；any_layer（任一层）才是局部 collapse 的正确聚合。

## 8. 边界
- toy 2 层 MLP，不等于真实大模型（几十层）。consecutive 在真实多层可能适用，需重新评估。
- collapse 首现深层（hidden_1）是 toy 现象（wd5 强正则），真实模型 collapse 形态可能不同（某中间层、或全层渐变）。
- 阈值（0.9/1.5）与聚合规则都是 benchmark-specific，不宣称适用真实大模型。
- 不改 CLI / 核心 SOAP，multilayer 是独立诊断模块。
