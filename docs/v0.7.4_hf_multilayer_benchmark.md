# SOAP-Core v0.7.4 Benchmark 科学文档

## 摘要

本文档记录 SOAP-Core 项目 v0.7.4 版本的基准测试结果。该测试旨在将 v0.7.3 在简单两层模型（any_layer 4/4 正确）上的 **representation collapse** 检测能力，外推验证至预训练多层 Transformer 模型（`distilbert-base-uncased`，6 层，`hidden_size=768`）。实验核心是验证检测器在深层网络中的**逐层误报率（FPR）** 与可控诱导坍缩的**传播路径分析**。v0.7.4 同时系统性地记录了方法探索过程中的关键负面结果，为理解有效检测指标奠定了基础。

## 证据边界（务必先读）

- 本 benchmark 的 collapse 为 **人工 controlled rank-1 intervention**，验证的是检测器与传播分析，**不代表自然训练 mode collapse 已被验证**。
- 实验对象为 **预训练 DistilBERT 在合成（synthetic）5 类分类任务上的微调 run**（非自然语料、非生产任务），结论限于该模型/任务/数据规模，**不外推到任意真实模型或自然训练动力学**。
- 误报率基于 **3 seed 的 leave-one-seed-out**，小样本：`0.333` 即 **1/3 held-out normal seed 误报**，**不外推为总体百分比**。
- 下文区分三种 FPR，**不可混用**：per-layer FPR（单层独立）/ run-level any_layer FPR（任层任 checkpoint 命中即记一次）/ persistent FPR（同层连续 ≥3 checkpoint 命中）。

## 实验设置

- **模型**：`distilbert-base-uncased`。冻结 Embedding 层和前 3 个 Transformer Block（`layer_0` - `layer_2`）。可训练部分为 `block 3` 到 `block 5`（对应输出层 `layer_4` 到 `layer_6`）及一个简单的分类头（Mask 加权 Mean Pooling + `Linear(768, 5)`）。
- **数据**：5 类结构化英文模板文本。训练集 1600 样本，验证集 400 样本，探针集（probe set）固定为 128 样本（`data_seed=0` 固定）。实验 `seed` 仅控制模型初始化、Dropout 和 Batch 顺序。
- **训练**：共 120 步，Batch Size 16，AdamW 优化器（`lr=2e-5`, `weight_decay=0`）。每 3 步在固定探针集上记录一次指标，共计 40 个 Checkpoint。
- **随机种子**：使用 `seed` 42, 43, 44。
- **实验条件**：
    - **Normal**：标准的分类任务训练。
    - **Induced Collapse**：在训练中引入**受控的渐进式 rank-1 激活干预**。
- **几何指标**：在固定探针集的 `pooled` 表征（形状 `(128, 768)`）上计算：
    - `effective_rank`：基于特征值熵的有效秩，衡量能量分布的集中度。
    - `representation_variance`：表征方差。
    - `collapse_score`：定义为 `1 / effective_rank`，值越高表示坍缩程度越严重。

### 复现信息
- 模型：`distilbert-base-uncased`，revision `12040accade4e8a0f71eabdb258fecc2e7e948be`
- 依赖：transformers 5.12.1，torch 2.12.1
- 设备：MPS（Apple Silicon）；**不宣称跨设备逐位可复现**，仅记录实际 device

## Controlled Rank-1 Activation Intervention（受控干预方法）

本实验采用一种**人工设计的干预手段**，用于在可训练层诱发可控的、可追踪的 **representation collapse**，以验证检测器的性能。**这并非自然训练过程中产生的坍缩**。

- **干预位置**：在 `block 3` 的输出（即 `hidden_states[4]`，模型中第一个可训练层的输出）施加 rank-1 投影。
- **干预公式**：在每个 Batch 内，于样本池化（pooled）空间构造 rank-1 目标，并将其修改量广播地加回原始 hidden states。
    ```math
    pooled_i = masked_mean(hidden_i)            # (B,768) 每样本池化表征
    global_center = mean(pooled, dim=0)         # 跨样本的全局中心
    centered = pooled - global_center
    target = global_center + (1-α)·centered + α·((centered·u)·u)   # 当 α→1 时，能量向 u 集中
    correction = target - pooled
    modified_hidden = hidden + correction（广播到 token 维，仅加到非 pad 位）
    ```
- **方向 `u`**：使用初始探针集池化表征的 PCA 第一主方向，在整个训练运行期间固定，不依赖于 induced 结果。
- **Alpha 调度**：`alpha` 从 0 开始，`step 0-20` 为 0（无干预），`step 21-120` 线性增加至 0.98。Normal 条件下 `alpha` 始终为 0。
- **训练目标**：仅使用分类损失（无额外的 contraction loss 或 λ 系数）。

## 方法论负面结果（关键探索过程）

在确定最终干预方案前，经历了 4 轮失败的干预设计。这些负面结果揭示了有效坍缩检测与干预的核心：**干预必须使能量集中到少数方向，并且其作用空间必须与检测指标（`effective_rank`）的计算空间（样本池化空间）一致**。

1.  **失败 1：`-log(var)` Contraction Loss**
    - **设计**：添加 loss `-log(var)` 试图直接最小化方差。
    - **问题**：方向错误。最小化 `-log(var)` 实际是鼓励方差 `var` 增大，与预期相反。
2.  **失败 2：Token-level `log(var)`**
    - **设计**：在 token 维度计算方差并添加 loss。
    - **问题**：度量空间不一致。token 维的收缩不会直接转化为样本池化空间（sample维）的 `effective_rank` 下降。
3.  **失败 3：Sample-level `log(var)`**
    - **设计**：在样本池化空间添加 loss 惩罚方差。
    - **问题**：该 loss 倾向于让所有方向的方差均匀缩小（各向同性收缩），但 `effective_rank` 是归一化的能量集中度，各向同性收缩后归一化后的集中度（熵）可能反而增大。
4.  **失败 4：Rank-1 Token-centered**
    - **设计**：使用每个样本自身的 token 均值进行中心化，再施加 rank-1 投影。
    - **问题**：保留了每个样本均值的样本间满秩差异，导致跨样本的池化表征 `effective_rank` 无法有效降低。
5.  **成功方案（方案 5）**：如“受控干预方法”所述，在**样本池化空间**使用**跨样本的全局中心**进行中心化并构造 rank-1 目标，再写回 hidden states。这保证了干预直接作用于检测指标所在的空间。

## 校准方法与评估规则

- **阈值校准**：采用 **Leave-One-Seed-Out** 交叉验证。每一折使用其余两个 `normal` seed 的数据，计算每个网络层（`layer_0` 到 `layer_6`）的 `collapse_score` 序列，并取第 99 百分位数作为该层的检测阈值。阈值校准**仅使用 normal 数据**，严禁使用 induced 数据反向调参。
- **检测规则**：
    - **any_layer**：在任意 checkpoint，任意网络层的 `collapse_score` 超过对应层阈值，即判定检出。
    - **persistent**：在同一网络层，连续 ≥3 个 checkpoint（即 9 个训练步）的 `collapse_score` 超过该层阈值，即判定检出。
- **误报率计算**：在 held-out 的 `normal` seed 上，应用上述规则，统计触发检出的 seed 比例。

## 结果

### 逐层检测阈值（Normal 数据第 99 百分位数）
| 网络层 (Layer) | 阈值 (collapse_score) |
| :--- | :--- |
| `layer_0` (冻结) | 0.046 |
| `layer_1` (冻结) | 0.061 |
| `layer_2` (冻结) | 0.063 |
| `layer_3` (冻结) | 0.069 |
| `layer_4` (可训练) | 0.106 |
| `layer_5` (可训练) | 0.180 |
| `layer_6` (可训练) | 0.202 |

### Induced Collapse 检出性能
- **检出率**：3/3 seeds（42, 43, 44）均被成功检出（`any_layer` 和 `persistent` 规则均满足）。
- **首次检出层**：均为 `layer_4`，首次检出步长均为 `step_102`。
- **坍缩传播路径**：三 seed 完全一致，均为 `layer_4` → `layer_5` → `layer_6`。
- **区分度**：
    - `layer_4`: induced 坍缩峰值 `collapse_score` = 0.974，约为阈值 (0.106) 的 **9 倍**，区分度强。
    - `layer_5`: `collapse_score` ≈ 0.24，略高于阈值 (0.180)。
    - `layer_6`: `collapse_score` ≈ 0.23，略高于阈值 (0.202)。

### 误报率 (FPR) - Leave-One-Seed-Out, Normal 数据（3 seed 小样本）

> 三种 FPR 含义不同，**不可混用**：per-layer FPR = 单层独立统计；run-level any_layer FPR = 该 run 任一层任一 checkpoint 命中即记 1 次；persistent FPR = 同层连续 ≥3 checkpoint 命中即记 1 次。下值均为 3 折 leave-one-seed-out 的平均，`0.333` 即 **1/3 held-out normal seed 误报**，**不外推**。

- **per-layer FPR**（单层独立）：
    - `layer_0` - `layer_3`（冻结）: **0.000**（0/3）
    - `layer_4`, `layer_5`, `layer_6`（可训练深层）: **0.333**（各 1/3 seed 误报）
- **run-level any_layer FPR**: **0.333**（1/3 held-out seed 触发）
- **persistent FPR**: **0.333**（1/3 held-out seed 触发）

## 结论

1.  **controlled 坍缩的可检测性（仅限本 benchmark）**：在当前 controlled rank-1 benchmark 内，检测器对人工 induced collapse 表现出可检测性：3/3 检出、首现层 = 施加层 `layer_4`、三 seed 一致传播路径 `layer_4→5→6`，`layer_4` 约 9 倍阈值区分度。**此结论限于 controlled rank-1 intervention，不代表自然 mode collapse 已被验证**。
2.  **层数增加暴露误报问题**：与 v0.7.3 的 2 层 toy 模型不同，**预训练 DistilBERT 合成任务 normal run** 的深层（`layer_4/5/6`）在训练后期 `effective_rank` 自然下降，`collapse_score` 持续超过阈值 → **深层 per-layer FPR = 0.333（1/3 held-out seed 误报，小样本不外推）**。`persistent` 规则**未能降低**误报（run-level any_layer FPR 与 persistent FPR 同为 1/3），表明该设置下深层的低秩倾向是持续动态而非瞬时波动。这是 v0.7.3 toy 实验未能预见的局限性，**仅限本模型/任务设置**。
3.  **干预特异性验证**：冻结层（`layer_0-3`）的 FPR 为 0，证实了 `block 3` 输出的干预仅影响其下游的可训练层，未反向污染冻结层，验证了干预机制的空间特异性。
4.  **坍缩传播的非完全性**：`layer_5` 和 `layer_6` 的 `collapse_score` 虽超阈值，但远低于 `layer_4` 的 rank-1 水平，表明 `block 4` 和 `block 5` 作为可训练层，具备一定的**表征重扩张能力**，使得人为诱发的坍缩无法完全传播至最深层。这本身是一个有价值的发现。

## 验收约束与边界说明

- **数据纯洁性**：检测阈值仅使用 `normal` 数据校准。
- **实验重复性**：所有关键结论基于 ≥3 个独立随机种子。
- **干预明确性**：本文档明确标注实验中的坍缩为**受控人工干预（controlled intervention）**，用于验证检测器与传播分析，**不代表自然训练过程中产生的坍缩**。
- **方法透明**：完整保留并展示了关键的**方法论负面结果**，以明确有效干预的设计空间。
- **操作范围**：本版本代码变更仅涉及本地 benchmark 脚本与文档，不涉及核心库的推送（只 diff 不 push）。
- **阈值泛化**：v0.7.3 在 toy 模型上得出的阈值结论不可直接固化应用于真实模型。针对不同模型架构、任务和数据集，需按本文所述的**逐层、按任务**方法重新校准阈值。
