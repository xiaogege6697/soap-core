# SOAP-Core v0.6 Training Adapter Notes

日期：2026-06-26

---

## 1. 本轮目标

v0.6 启动**应用适配层（application adapter）**，首个落地场景为 AI training run 不稳定性检测。

本轮工作范围：

- 仅实现数据适配（synthetic generator + loader adapter）
- 不修改核心算法
- 不接入 CLI

设计原则：核心层保持领域无关，适配层负责将领域数据转换为核心层可消费的标准格式。

---

## 2. 为什么选 AI training instability 作为第一个应用场景

1. **数据天然存在**：训练过程会持续产出 `loss`、`grad_norm`、`learning_rate`、`val_loss` 等遥测指标，无需额外埋点，直接可用。
2. **决策闭环短**：检测到异常后可立即执行「暂停训练 → 降 learning rate → 回滚 checkpoint」等操作，效果可量化。
3. **竞品空白**：W&B / TensorBoard 偏向 dashboard 可视化与固定阈值告警，缺少对状态空间结构的解释能力；SOAP 在 attractor 结构与 recurrence 分析上有差异化空间。
4. **可回放验证**：历史 training run 数据可用于离线回放，验证 SOAP 能否比简单阈值法更早识别 divergence / overfit / mode collapse。

---

## 3. 数据字段

| 字段名 | 含义 | 典型量级 |
|---|---|---|
| `timestamp` | 步骤时间戳（Unix epoch，秒） | 10⁹ 量级 |
| `loss` | 训练 loss | 0.01 – 10.0 |
| `grad_norm` | 梯度范数 | 0.1 – 100.0（发散时可达 10³+） |
| `learning_rate` | 当前学习率 | 1e-6 – 1e-1 |
| `val_loss` | 验证集 loss | 0.01 – 10.0 |

> 注意：合成数据量级基于常见 Transformer / ResNet 训练经验，真实场景应根据模型规模调整。

---

## 4. 模块说明

### `soap/apps/training/generator.py`

合成训练日志生成器。

- **`normal` 模式**：模拟稳定收敛曲线，loss 从高位单调下降并趋于平稳，`grad_norm` 逐渐收敛。
- **`diverging` 模式**：前期行为与 `normal` 一致，后期注入发散信号——`grad_norm` 指数上升、`loss` 震荡飙升。
- **可复现**：所有随机过程绑定固定 `seed`，相同参数产出相同 CSV。

### `soap/apps/training/adapter.py`

- **`load_training_run(path)`**：内部复用 `soap.data.loader.load_csv`，返回与核心 loader 一致的类型（DataFrame），调用方无需感知训练领域细节。
- **`describe_training_run(path)`**：返回字段名列表与行数，用于接入前的轻量校验，避免脏数据进入核心流程。

## 5. 示例用法

### 生成合成数据

```python
from soap.apps.training.generator import generate_training_run

# 生成 5000 步的稳定收敛日志
generate_training_run(mode="normal", steps=5000, seed=42, output="data/normal_run.csv")

# 生成第 3000 步开始发散的日志
generate_training_run(mode="diverging", steps=5000, divergence_at=3000, seed=42, output="data/diverging_run.csv")
```

### 适配载入与标准化

```python
from soap.apps.training.adapter import load_training_run
from soap.core import standardize  # 假设的核心标准化接口

df = load_training_run("data/normal_run.csv")
result = standardize(df)  # 交由核心层处理，返回状态空间分析结果
```

> ⚠️ 当前未接入 CLI，仅支持 Python API 调用。

## 6. 后续 benchmark 思路

1. **数据集构建**：收集真实 training run 或使用合成数据，人工标注 divergence 起始步（ground truth）。
2. **对比基线**：将 SOAP 识别出的异常点与固定阈值法（如 `grad_norm > 100`）的命中时间进行对比，验证是否能**更早**识别。
3. **结构差异分析**：对比 `normal` vs `diverging` run 在以下维度上的差异：
   - **有效维度（effective dimension）**：发散时状态空间是否坍缩或膨胀
   - **吸引子结构（attractor structure）**：收敛轨迹的 attractor 是否稳定
   - **递归特征（recurrence）**：发散前是否出现 recurrence pattern 的突变

## 7. 边界声明

| 约束 | 说明 |
|---|---|
| 范围 | v0.6 **仅**应用适配层，不涉及核心算法改动 |
| 数据真实性 | generator 产出为合成数据，**非**真实训练日志 |
| CLI | 未接入，仅提供 Python API |
| 核心依赖 | 不修改 `soap.core` 与 `soap.data.loader` |
